# Code of Conduct

## Our Goal

We want this project to feel welcoming, practical, and respectful.

Discussion should help people build, learn, and collaborate. Curiosity is good. Personal attacks are not.

## Expected Behavior

- Be respectful when giving feedback
- Focus on the code and the problem, not the person
- Assume good intent before escalating
- Share enough context for others to reproduce issues
- Keep security reports private until a fix is ready

## Unacceptable Behavior

- Harassment, insults, intimidation, or discrimination
- Publishing private information without permission
- Trolling, baiting, or intentionally derailing technical discussion
- Posting exploit details for an unfixed vulnerability in public

## Enforcement

Project maintainers may edit, hide, lock, or remove content that breaks these rules.

Severe or repeated abuse can lead to being blocked from further participation.
