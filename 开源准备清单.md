# QQ Bot 开源前准备清单

本文档列出了将当前 QQ 机器人项目开源至 GitHub 前必须完成的各项准备工作，旨在确保项目安全、易用且专业。

## 1. 核心安全与防泄露 (最高优先级)

- [x] **创建 `.gitignore` 文件**
  - 必须包含：`.env`、`__pycache__/`、`data/`（用户上传图片、聊天记录数据库等）。
  - 可选包含：本地 IDE 配置（如 `.idea/`、`.vscode/`）、系统生成文件（如 `.DS_Store`）。
- [x] **脱敏环境变量**
  - 将真实的 `.env` 文件保留在本地，不可提交。
  - 创建 `.env.example` 模板文件，去掉真实 Key，填入占位符（如 `LLM_API_KEY=your_api_key_here`）。
- [x] **代码硬编码清理**
  - 检查 `src/` 目录和 `bot.py`，确保没有写死的“管理员 QQ 号”、“测试群群号”。
  - 检查文件路径，确保没有写死的本地绝对路径（如 `C:\Users\...`），应改为基于项目根目录的相对路径或动态获取路径。
  - 已将公开文档中的本机路径改为通用写法，并将文生图工作流改为 `workflows/text2img_default.json`。

## 2. 项目基础信息与文档门面

- [x] **完善项目首页 `README.md`**
  - **项目简介**：一句话说明项目的功能与特色（LLM 对话 + ComfyUI 生图）。
  - **功能特性列表**：提炼 `PRD.md` 中的核心亮点。
  - **环境依赖**：说明 Python 版本要求（如 Python 3.10+），以及所需的第三方服务（如 NapCatQQ、ComfyUI）。
  - **快速开始 (Quick Start)**：
    - 克隆仓库的命令。
    - 安装依赖的命令 (`pip install -r requirements.txt`)。
    - 配置环境的说明。
    - 启动项目的命令 (`python bot.py`)。
  - **展示效果图**：添加几张机器人在群聊中对话和生图的截图（非常重要，吸引用户）。
- [x] **添加开源许可证 (License)**
  - 在项目根目录添加 `LICENSE` 文件（推荐使用 MIT 协议，对使用者最友好且限制最少）。
- [x] **梳理现有文档**
  - 已将文生图工作流整理为 `workflows/text2img_default.json`，并在文档中说明如何使用。
  - 更新 `使用教程.md` 或将其部分核心内容整合进 `README.md` 中。
