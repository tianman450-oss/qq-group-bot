# QQ 群聊 & 生图机器人 PRD (产品需求文档)

## 1. 项目概述
开发一个接入 QQ 群聊的智能机器人。核心功能包括两块：基于大语言模型（LLM）的智能对话，以及基于 ComfyUI 的 AI 图像生成。为 QQ 群成员提供便捷的问答、闲聊以及 AI 绘画服务。

## 2. 核心功能需求

### 2.1 QQ 基础接入层
- **指令监听与回放**：监听群聊文本、图片、At（@）消息，支持以文本、图片、引用回复（Reply）等格式回应。
- **权限与频率控制**：
  - 支持群组白名单/黑名单。
  - 防刷机制（如聊天 CD 限制、生图单人每日限额）。
- **触发方式**：
  - @机器人 或 指定前缀（如 `!` 或 `/`）。

### 2.2 大模型聊天功能 (LLM API)
- **多轮对话能力**：维护会话上下文（短期记忆），支持根据 User ID 或 Group ID 隔离会话，设定记忆轮数（如 5-10 轮）或过期时间（如 10 分钟无交互则清空）。
- **角色设定 (System Prompt)**：可配置机器人的 Prompt 人设与性格。
- **超时与兜底**：遇到大模型触发风控或 API 超时，回复友好的提示语。

### 2.3 ComfyUI 生图功能
- **指令唤醒**：如发送 `/画 [描述]`, `/draw [描述]` 触发。
- **Prompt 预处理**：大模型通常配合使用，可通过 LLM 先将用户的中文描述优化、翻译为高质量的英文 Prompt，再发送给 ComfyUI。
- **ComfyUI 接口交互**：
  - 调用 ComfyUI 的 `/prompt` HTTP/WebSocket 接口执行生图 Workflow。
  - **任务队列控制**：生图极度消耗 GPU 资源，必须实现**严格的排队机制**。在并发请求时，提示用户“当前排队 X 人，请稍候...”。
- **高级参数（可选暴露）**：解析指令中的横竖屏指令（如 `--ar 16:9` 控制传入 Workflow 的图片宽高）。
- **结果回传**：图片生成后通过 QQ 协议发送原图/压缩图，并可加上生成耗时。

### 2.4 进阶应用功能
- **联网搜索**：为大模型赋予搜索能力，解决知识库时效性问题，获取最新资讯与实时数据。
- **群聊消息总结**：支持指令触发（如 `/总结` 或 `@机器人 今天聊了啥`），提取群内近期聊天记录由大模型生成摘要，帮助成员快速回顾上下文。
- **图生图与图像处理 (ComfyUI 扩展)**：除了文生图，支持接管用户发送的图片进行再创作（如照片转二次元、线稿一键上色、老旧低清表情包的高清超分放大）。
- **长期记忆与用户画像**：引入轻量级数据库（如 SQLite），实现超越单次会话的长期记忆，记住特定群友的称呼、习惯与偏好设定。
- **定时与推送服务**：支持配置定时任务，例如每天定时发送“早报新闻”或“每日摸鱼图”到指定群组。
- **大学生课表同步与上课提醒**：
  - **WakeUp 课程表口令导入**：全面接入「WakeUp课程表」App。用户在 App 内生成分享口令（如 `493f666c20f54f4f9dd0f19ad614f228`）发送给机器人，机器人即可提取口令，通过接口抓取并同步用户的全量周课表数据及作息时间表。
  - **精准定位当前课程**：导入后，机器人能计算并知晓用户当前时刻（精确到单双周和节次）是否正在上课以及课程名称。
  - **全局被动提醒**：当用户处于上课时间，无论是与机器人聊天，还是触发**其他所有功能指令**（如生图、搜索），机器人均会在回复的最前面或上下文里，自然顺滑地附带符合角色人设的劝学提醒（如“怎么上课还在鼓捣画图呀？好好听讲！”），再执行并回复用户的原有请求。该提醒由大模型根据情境动态发挥，拒绝机械文本。

## 3. 非功能及架构需求
- **容错稳定性**：任意环节出错（LLM 调用断联、ComfyUI OOM 显存溢出）均需被 try-catch 捕获，绝不能导致主进程崩溃，且要给群内反馈错误。
- **持久化配置**：使用 `.env` 或 YAML 文件管理 API Key、ComfyUI 地址、QQ 账密配置等。

## 4. 技术栈选型推荐
- **底层通信协议**：**NapCatQQ** 或 **Lagrange** (实现 OneBot V11 协议，目前封号率低，适配性好)。
- **开发框架**：**NoneBot2** (Python 生态，编写逻辑极简，支持异步处理，非常适合本场景)。
- **网络请求**：`httpx` / `websockets` 用于对话与生图。

## 5. 架构选型与部署计划
- **部署环境**：云端服务器。服务端运行 NoneBot2 业务逻辑 + NapCatQQ（无头/Linux 版）负责 QQ 消息收发。
- **LLM 接入**：DeepSeek / Qwen（通义千问），均兼容 OpenAI 格式的 API 接口。
- **生图引擎**：采用已有 ComfyUI 导出的 API JSON 工作流。
